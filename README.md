# Парсер Wildberries

Собирает товары с Wildberries по запросу «пальто из натуральной шерсти».

Написан на **Scrapy**. Сначала сохраняет данные в `json`, потом из них собираются два XLSX-файла.

## Конфигурация

Используется одна переменная окружения:

- `X_WBAAS_TOKEN` — токен WB для поискового запроса

Файл `.env` размещается рядом с `scrapy.cfg`:

```env
X_WBAAS_TOKEN=<token>
```

Пример заполнения есть в `.env.example`.

## Что собирает

| Поле                  | Описание                                   |
| --------------------- | ------------------------------------------ |
| Ссылка на товар       | URL карточки на wildberries.ru             |
| Артикул               | Числовой ID товара (nm_id)                 |
| Название              | Наименование товара                        |
| Цена (руб.)           | Цена                                       |
| Описание              | Текстовое описание из карточки             |
| Ссылки на изображения | Все фото товара через запятую              |
| Характеристики        | Все характеристики (структура сохраняется) |
| Название селлера      | Имя продавца                               |
| Ссылка на селлера     | URL страницы продавца                      |
| Размеры               | Доступные размеры через запятую            |
| Остатки               | Суммарный остаток                          |
| Рейтинг               | Средний рейтинг товара                     |
| Количество отзывов    | Число отзывов                              |

## Результат

- `data/products_YYYY.MM.DD_HH-MM-SS.json` — сырые данные после парсинга
- `output/catalog.xlsx` — полный каталог
- `output/filtered.xlsx` — выборка: рейтинг ≥ 4.5, цена ≤ 10 000 руб., страна производства — Россия

## Установка

```bash
pip install -r requirements.txt
```

## Запуск

```bash
scrapy crawl wildberries
```

Ограничение по числу страниц (по умолчанию 5):

```bash
scrapy crawl wildberries -a max_pages=10
```

После парсинга конвертация в XLSX:

```bash
python converter.py
```

По умолчанию скрипт берёт самый свежий `json` из папки `data/`.
Запуск с явным указанием файла:

```bash
python converter.py data/products_YYYY.MM.DD_HH-MM-SS.json
```

## Как работает

1. `start_requests` — сначала получает список basket-хостов с CDN WB.
2. `parse_first_page` — запрашивает первую страницу поиска и считает, сколько страниц нужно собрать.
3. `parse_search` — берёт базовые поля из поиска.
4. `parse_card` — для каждого товара догружает описание и характеристики из `card.json`.
5. Scrapy через `FEEDS` сразу сохраняет результат в `json`.
6. `converter.py` читает `json` и создает `catalog.xlsx` и `filtered.xlsx`.

Basket берётся из CDN-эндпоинта WB по диапазону `vol`.
Поиск использует токен `X_WBAAS_TOKEN` из `.env`.
Логи пишутся в папку `logs/`.
